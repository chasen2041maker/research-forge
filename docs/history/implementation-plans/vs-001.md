# Baseline Reproduction Vertical Slice

> Historical implementation plan. For the supported runtime subset, use
> [runtime-capability-profile-v0.1.md](../../contracts/runtime-capability-profile-v0.1.md).

> Slice：VS-001
>
> 决策：第一条新架构链路完全不使用 LLM、LangGraph、Skill、MCP、Reviewer、Writer 和 UI。
> 目标：先证明状态、恢复、Git、CAS、沙箱和指标真相正确。

## 1. 固定场景

输入 Fixture：

```text
tests/fixtures/toy_reproduction_repo
```

冻结参数：

```yaml
mode: reproduce
repository_commit: <fixture-full-sha>
image_digest: <prebuilt-image-digest>
run_argv:
  - python
  - evaluate.py
  - --output
  - metrics.json
metric_pointer: /accuracy
expected_value: 0.80
tolerance: 0.001
network: offline
candidate_changes: 0
```

完整输入必须通过 [ReproductionSpec v1](../../contracts/reproduction-spec-v1.md) Validator。

## 2. 不包含的能力

- 不理解任意论文；
- 不生成计划；
- 不修复代码；
- 不创建 Candidate Worktree；
- 不调用模型；
- 不写自然语言研究报告；
- 不连接 MCP/GitHub；
- 不发送 Draft PR；
- 不做复杂前端。

## 3. 端到端流程

```text
CreateMission
→ Validate ReproductionSpec
→ Create Baseline Task/Attempt
→ Enqueue Outbox
→ Worker Claims Lease
→ Create Baseline Worktree
→ Create Sandbox Operation
→ Execute Fixed Command Offline
→ Put Log/Metric into CAS
→ Register Artifact/Metric
→ Deterministic Metric Validation
→ Create Verified Result Claim
→ Build Deterministic Bundle
→ Mission Completed
```

## 4. 状态

Mission：

```text
DRAFT → READY → RUNNING → VERIFYING → COMPLETED
```

失败：

```text
READY/RUNNING/VERIFYING → FAILED
```

取消：

```text
RUNNING → CANCELLING → CANCELLED
```

Task：

```text
BASELINE_REPRODUCTION
```

Attempt：

```text
PENDING → RUNNING → SUCCEEDED
                 ├→ RETRYABLE
                 ├→ FAILED
                 └→ CANCELLED
```

## 6. Use Cases

### `CreateReproductionMission`

输入：ReproductionSpec v1。

输出：Mission View。

事务：

- 创建 Mission；
- 创建 Task；
- 创建 Attempt；
- 写 Audit；
- 写 Outbox。

### `RunBaselineAttempt`

输入：Attempt ID。

输出：Operation/Execution Proposal，由 Application 直接根据 Spec 确定，不需要 Agent。

### `FinalizeBaselineExecution`

输入：Sandbox Result、Artifact References、Lease Epoch。

输出：Metric Validation Result。

事务：

- Finalize Operations；
- 注册 Artifact/Metric；
- 条件更新 Attempt；
- 写 Audit/Outbox。

### `CompleteReproductionMission`

输入：已通过的 Metric 和 Evidence Closure。

输出：Bundle Artifact Ref。

## 8. Operation Ledger 故障点

必须注入 Kill：

1. `PREPARED` 后、执行前；
2. Sandbox 完成后、CAS 写入前；
3. CAS Rename 后、DB Finalize 前；
4. Git Worktree/Ref 创建后、DB Finalize 前；
5. DB Finalize 后、Queue ACK 前。

每次恢复必须满足：

- 一个逻辑 Operation；
- 一个正式 Artifact Registration；
- 一个成功 Attempt；
- 无重复 Commit/Ref；
- 无重复 Metric；
- 孤儿 Blob 可被 GC 回收。

## 12. 退出条件

- Baseline Demo 连续运行 10 次；
- Kill/Resume 不产生重复副作用；
- Artifact 篡改可检测；
- 旧 Worker 不能提交；
- 执行阶段网络为零；
- Bundle 可在新目录重放；
- Architecture CI 无违规。

未满足退出条件前，禁止进入第二条 `repair` Slice。
