# v0.2 roadmap

The following are plans, not shipped capabilities:

1. Replace bare Redis Attempt-ID transport with versioned Redis Streams, routing, visibility
   timeout, and a dead-letter queue.
2. Bind patch bytes and approvals more tightly, then add independent external-fixture evaluation.
3. Complete the Studio/Forge product loop with a read-only verified-result report in Studio.

No roadmap item changes the current rule: Studio proposes, Forge verifies.
