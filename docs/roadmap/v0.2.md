# v0.2 roadmap

The following are plans, not shipped capabilities:

1. Complete the Task, Operation, and Approval concurrency contract.
2. Persist durable sandbox-broker result state across broker restart.
3. Replace bare Redis Attempt-ID transport with versioned Redis Streams, routing, visibility
   timeout, and a dead-letter queue.
4. Bind patch bytes and approvals more tightly, then add independent external-fixture evaluation.
5. Complete the Studio/Forge product loop with a read-only verified-result report in Studio.

No roadmap item changes the current rule: Studio proposes, Forge verifies.
