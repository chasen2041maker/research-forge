# 新项目想法 · 索引与选型决策

> 三份方案各自独立可落地。这一页是选型指南 —— **先读这个,再进具体方案**。

## 一、我为什么需要"下一个项目"

当前的 AI Co-Scientist(agent3)**作为升级版毕业设计够用**,但有两个硬伤:
1. **虚构场景**:没有真实用户,没有真实使用数据
2. **架构已定型**:继续加模块是"加法",不再体现"架构决策能力"

2025-2026 招聘市场对 Agent 工程师的期望:
- **真实用户数据 / 产品感**(不是 Demo)
- **紧跟前沿架构**(MCP / Orchestrator-Subagent / Long-running)
- **至少一个可讲的开源/部署作品**

所以下一个项目的目标是**补齐其中至少一项**。

## 三、决策树

```
你离找工作还有多久?
├─ < 2 个月 → 只做 A(时间不够做别的)
├─ 2-4 个月 → A + B1(双项目,互补叙事)
└─ > 4 个月 → A(热身) + C(深做)
                    或
                   B(深做) + A(辅助)

你想进什么岗位?
├─ 中小厂 AI 应用 → A 足够
├─ 大厂 AI 应用 / 头部独角兽业务岗 → A + B 组合
├─ 大厂 AI Infra / Agent 基建团队 → C(必须) + A(展示架构理解)
└─ 研究院 / AI Lab → C(开源 + 论文阅读量) + A(展示工程力)

你能接受的风险?
├─ 低风险、稳健提升 → A
├─ 中风险、收益更高 → B(真实用户可能爱/可能不爱)
└─ 高风险、赌未来 → C(开源可能没人用,但一旦爆就起飞)
```

## 四、推荐路径(按多数人情况)

### 🥇 最稳妥路径:A(Week 1-3)→ B1(Month 2-2.5)

1. **Week 1-3**:按 `A_当前项目做深.md` 升级 Co-Scientist 到 2026 前沿
2. **Week 4**:写一篇博客总结架构演进,放 GitHub / 个人站
3. **Month 2-2.5**:新起 B1 个性化学习 Agent,**复用**已升级过的 evolve/memory/evals 模块
4. **完成后**:简历上两个项目 —— "架构深度项目"+"场景落地项目",互补不重复

### 🥈 时间紧(<2 月)路径:只做 A

只做 A 也完全能讲,**关键是做深**:MCP 集成 + 博客 + 在线 Demo 三件事做完,架构档次直接跳两级。

### 🥉 目标高(头部基建岗)路径:A(热身)→ C1(深做)

A 花 2 周快速升级,作为 "I understand agent architecture" 的 Demo。
C1(MCP Gateway)花 2 月深做,作为 "I build agent infrastructure" 的开源作品。
