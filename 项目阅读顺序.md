## Part 1：文档阅读顺序（先看这条线）

目标：1~2 小时内对整个项目"在做什么、为什么这么做、有哪些核心模块"形成完整心智模型，再去碰代码。

### 1. `README.md` — 5 分钟
- 项目一句话介绍、技术栈、目录结构
- 跳过细节，只看"它能干什么"和"目录布局"

### 2. `AI-Co-Scientist-技术方案.md` — 1 小时（重点）
这是整个项目的设计圣经，按下面顺序看，不要从头到尾顺读：

| 顺序 | 章节 | 看什么 |
|---|---|---|
| ① | §1 总体架构 | 8 个模块 + 数据流图，建立总体感 |
| ② | §2 模块清单 | 每个 m1~m8 一句话职责 |
| ③ | §3 关键设计决策 | "为什么选 LangGraph / DeepSeek+Claude 双模型 / SQLite Checkpointer" |
| ④ | §4~§11 各模块设计 | 按 m1 → m7 顺序读，每节看完想想"我会怎么实现"再看下一节 |
| ⑤ | §附录 A | 自我进化（记忆库 + Prompt A/B）— 已在本项目挂入主流程 |
| ⑥ | §附录 B | 对抗式数据工厂（Red/Blue/Judge）— 已暴露为 CLI 子命令 |
| ⑦ | §附录 C | 成本控制策略（DeepSeek 优先版） |

> ⚠️ 看 §附录 A/B 时配合 Part 2 §F 一起，看"设计如何落到代码"。

- 项目的"教学路线图"，列出每个模块对应的 NLP/Agent 经典论文

### 4. `教学资料/` — 按需选读
按编号对应模块，写得比技术方案更口语：
- `01-基础工程` `02-LLM调用` `03-Agent框架` `04-RAG检索` `05-多Agent协作`(含 5.11 节 Orchestrator-Subagent)

- `10-Web集成`

### 5. `面试讲稿_终极版.md` — 30 分钟（可选）
- 把项目当作面试材料的浓缩版，反过来检验你看完前面是不是真理解了

```
state/cards.py          ⭐ Phase A:整理版新增数据结构契约

                              GapCard(M3)/ DecisionCard(M4)
utils/logger.py            ← loguru 包装
utils/cost_tracker.py      ← LLM 调用计费记账(add() 里挂 budget hook),
```

### B. LLM 抽象层（5 分钟）

```
llm/base.py                ← BaseLLM 抽象类 + Message 类型
llm/deepseek.py            ← OpenAI 兼容协议客户端

                              既能跑 DeepSeek 也能复用为 GPT 中转站客户端
llm/claude.py           ⭐ Phase 3:chat() 新增 thinking_budget 支持 Extended Thinking
```
